# Migrations (Alembic)

Schema is managed by [Alembic](https://alembic.sqlalchemy.org/), autogenerated from the SQLAlchemy models in [`../models.py`](../models.py) (ADR-0006). The DB URL comes from `.env` via `env.py` — never hard-coded here.

## Common commands

Run from `backend/` (or use the `make` targets from repo root):

```bash
uv run alembic upgrade head                          # apply all pending migrations  (make migrate)
uv run alembic current                               # show current DB revision      (make migrate-status)
uv run alembic history                               # list all revisions
uv run alembic downgrade -1                           # roll back one revision
uv run alembic revision --autogenerate -m "add x"    # generate a migration from model changes
```

## Workflow when changing the schema

1. Edit the models in `data/models.py`.
2. `uv run alembic revision --autogenerate -m "describe change"`.
3. **Read the generated migration** — autogenerate doesn't catch everything (e.g. special index ops, server-side defaults). Adjust if needed.
4. `uv run alembic upgrade head` and verify.

## Conventions

- **Never edit an applied revision.** Add a new one.
- The initial revision (`0001_initial_schema`) was hand-authored to match the models; later ones should be autogenerated and reviewed.
